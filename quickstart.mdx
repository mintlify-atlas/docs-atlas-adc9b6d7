title: "Get started with Beetle Engine in minutes"
description: "Clone the repository, generate Visual Studio project files with Premake5, and run your first Beetle application on Windows x64 in a few steps."
---

Beetle Engine uses [Premake5](https://premake.github.io/) to generate Visual Studio solution and project files from a single `premake5.lua` script. The steps below walk you from a fresh clone to a running application using either the **Sandbox** test app or the **Stag** editor.

<Note>
  Beetle Engine currently targets **Windows x64 only**. Visual Studio 2022 is required. macOS, Linux, and other platforms are not yet supported.
</Note>

<Steps>
  <Step title="Clone the repository">
    Open a terminal and clone the Beetle Engine repository:

    ```bash
    git clone https://github.com/Sam-xiexielaoshi/Beetle.git
    cd Beetle
    ```
  </Step>

  <Step title="Generate Visual Studio project files">
    Run Premake5 targeting Visual Studio 2022. You can either call `premake5` directly or use the provided helper script:

    <CodeGroup>

    ```bash premake5 directly
    premake5 vs2022
    ```

    ```bash using the helper script
    scripts\Win-GenProjects.bat
    ```

    </CodeGroup>

    This generates `Beetle.sln` along with `.vcxproj` files for the **Beetle**, **Sandbox**, and **Stag** projects, and all vendored dependencies.
  </Step>

  <Step title="Open the solution">
    Open `Beetle.sln` in Visual Studio 2022. The workspace is pre-configured with **Stag** as the startup project.
  </Step>

  <Step title="Choose a build configuration">
    Beetle supports three configurations. Select one from the Visual Studio toolbar before building:

    | Configuration | Purpose |
    |---|---|
    | **Debug** | Full symbols, assertions enabled. Use this during development. |
    | **Release** | Optimized build, symbols stripped. Use for performance testing. |
    | **Dist** | Distribution build. Fully optimized, no debug output. |

    For your first run, **Debug** is recommended.
  </Step>

  <Step title="Build and run">
    Press **Ctrl+Shift+B** to build the entire solution, then press **F5** (or **Ctrl+F5**) to run the startup project.

    - To run the **Stag** editor, right-click the `Stag` project in Solution Explorer and choose **Set as Startup Project**, then run.
    - To run the **Sandbox** test app, do the same for the `Sandbox` project.

    Build output lands in `bin/<Configuration>-windows-x86_64/<ProjectName>/`.
  </Step>
</Steps>

## How the entry point works

Every Beetle application follows the same two-step pattern: subclass `Beetle::Application` and implement the `CreateApplication()` factory function. The engine's `EntryPoint.h` calls `CreateApplication()` internally and owns the application lifecycle — you never write `main()` yourself.

Here is the complete entry point from the Sandbox project:

```cpp
#include <Beetle.h>
#include <Beetle/Core/EntryPoint.h>
#include "Sandbox2D.h"

class Sandbox : public Beetle::Application
{
public:
    Sandbox()
    {
        PushLayer(new Sandbox2D());
    }
    ~Sandbox() {}
};

Beetle::Application* Beetle::CreateApplication()
{
    return new Sandbox();
}
```

The Stag editor follows the same pattern, passing a window title to the base constructor:

```cpp
#include <Beetle.h>
#include <Beetle/Core/EntryPoint.h>
#include "EditorLayer.h"

namespace Beetle {
    class Stag : public Application
    {
    public:
        Stag() : Application("Stag")
        {
            PushLayer(new EditorLayer());
        }
        ~Stag() {}
    };

    Application* CreateApplication()
    {
        return new Stag();
    }
}
```

To create your own minimal application, implement `CreateApplication()` and push at least one `Beetle::Layer` onto the stack. The engine handles the run loop, event dispatch, and shutdown for you.
